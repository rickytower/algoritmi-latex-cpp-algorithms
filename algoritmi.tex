		int new_min = st.empty() ? new_elem : min(new_elem, st.top().second);
		st.push({new_elem, new_min});
	\end{lstlisting}
	\begin{lstlisting}[caption={rimuovere un elemento.}]
		int removed_element = st.top().first;
		st.pop();
	\end{lstlisting}
	\begin{lstlisting}[caption={trovare il minimo.}]
		int minimum = st.top().second;
	\end{lstlisting}Lo stack ha un comportamento LIFO.
	\subsubsection{Modifica della coda -- metodo 1}La coda ha un comportamento FIFO. Non memorizza tutti gli elementi: vengono memorizzati solo quelli necessari per determinare il minimo. La coda viene mantenuta in \textit{ordine non decrescente}\footnote{cioè $\forall i \in [0,\texttt{coda.length}()-1]\colon \texttt{coda[i]} \le \texttt{coda[i+1]}$} (il valore più piccolo sarà memorizzato nella testa). Il minimo effettivo deve sempre essere mantenuto nella coda. 
	\begin{lstlisting}[caption={implementazione della coda.}]
		deque<int> q;
	\end{lstlisting}
	\begin{lstlisting}[caption={trovare il minimo.}]
		int minimum = q.front();
	\end{lstlisting}
	Prima di aggiungere un nuovo elemento nella coda è sufficiente effettuare un ``taglio'': vengono rimossi tutti gli elementi finali della coda che sono più grandi del nuovo elemento e successivamente si aggiunge il nuovo elemento alla coda. In questo modo viene mantenuto l'ordine non decrescente e non viene perso l'elemento corrente se in qualsiasi passaggio successivo è l'elemento minimo. Tutti gli elementi che vengono rimossi non potranno mai essere primi. 

	\begin{lstlisting}[caption={aggiungere un elemento.}]
			//dal fondo della coda si stanno rimuvendo tutti gli elementi di q maggiori di new_element
		while (!q.empty() && q.back() > new_element)
		q.pop_back();
		q.push_back(new_element);
	\end{lstlisting}
	
	Quando si vuole estrarre un elemento dalla testa, potrebbe non essere presente se è stato rimosso in precedenza con un elemento più piccolo. Pertanto quando si elimina un elemento da una coda bisogna conoscerne il valore. Se la testa della coda ha lo stesso valore, può essere rimossa in sicurezza, altrimenti non viene apportata nessuna modifica.
	\begin{lstlisting}[caption={rimuovere un elemento.}]
		if (!q.empty() && q.front() == remove_element)
		q.pop_front();
	\end{lstlisting}
	Tutte queste operazioni richiedono $O(1)$.

\end{lstlisting}

\subsubsection{Modifica della coda -- metodo 3}
Questo è un altro metodo per trovare il minimo in $O(1)$ modificando una coda. È più complicato da implementare ma permette di memorizzare tutti gli elementi e di rimuovere un elemento dalla parte anteriore senza conoscerne il valore. L'idea è di ridurre il problema al problema degli stack, simulando una coda. Si creano due pile, \texttt{s1} e \texttt{s2}. Si aggiungono nuovi elementi allo stack \texttt{s1}. e si rimuovono da \texttt{s2}. Se in qualsiasi momento \texttt{s2} è vuoto, si spostano tutti gli elementi da \texttt{s1} a \texttt{s2} (che inverte l'ordine degli elementi). Infine, per trovare il minimo in  una coda è sufficiente trovare il minimo di entrambi gli stack. \par
Si eseguono tutte le operazioni in $O(1)$ in media (ogni elemento verrà aggiunto una volta allo stack \texttt{s1}, una volta trasferito a \texttt{s2} e una volta estratto da \texttt{s2}).
\begin{lstlisting}[caption={implementazione.}]
	stack<pair<int, int>> s1, s2;
\end{lstlisting}
\begin{lstlisting}[caption={trovare il minimo.}]
	if (s1.empty() || s2.empty()) 
		minimum = s1.empty() ? s2.top().second : s1.top().second;
	else
		minimum = min(s1.top().second, s2.top().second);
\end{lstlisting}

\begin{lstlisting}[caption={aggiungi elemento.}]
	int minimum = s1.empty() ? new_element : min(new_element, s1.top().second);
	s1.push({new_element, minimum});
\end{lstlisting}

\begin{lstlisting}[caption={rimozione di un elemento.}]
	if (s2.empty()) {
		while (!s1.empty()) {
			int element = s1.top().first;
			s1.pop();
			int minimum = s2.empty() ? element : min(element, s2.top().second);
			s2.push({element, minimum});
		}
	}
	int remove_element = s2.top().first;
	s2.pop();
\end{lstlisting}

\subsubsection{Trovare il minimo per tutti i sottoarray di lunghezza fissa}
Si supponga di ricevere un array \texttt{A} di lunghezza $N$ e un dato $M$ tale che $M\le N$. Si deve trovare il minimo di ogni sottoarray di lunghezza $M$ in questo array, cioè si trovi: $$\min_{0\le i\le M-1}A[i],\min_{1\le i \le M} A[i], \min_{2 \le i \le M+1} A[i], \dots, \min_{N-M\le i \le N-1}A[i]$$
Questo problema deve essere risolto in tempo lineare, cioè $O(n)$.
\begin{figure}[tbph]
	\centering
	\includegraphics[width=0.7\linewidth]{sottoarray-stack-coda}
	\caption{sottoarray di dimensione $M$.}
	\label{fig:sottoarray-stack-coda}
\end{figure}

È possibile utilizzare una qualsiasi delle tre code modificate per risolvere il problema. 
\begin{enumerate}
	\item Seleziona la finestra.
	\item Aggiungi alla coda tutti gli elementi della finestra.
	\item Ricava il minimo.
	\item Rimuovi il minimo.
	\item Sposta la finestra in avanti di uno.
\end{enumerate}


 Poiché tutte le operazioni sulla coda richiedono tempo costante in media, la complessità complessiva dell'algoritmo risulta $O(n)$.

\subsection{Sparse table}
Una sparse table è una struttura dati che consente di rispondere a query di intervallo, rispondendo nella maggior parte dei casi in $O(\log n)$, a la sua vera potenza è rispondere a query con intervallo minimo/massimo in $O(1)$. L'unico inconveniente di questa struttura dati è che può essere utilizzata solo su \textit{matrici immutabili}. Ciò significa che la matrice non può essere modificato tra due query. Se un elemento della matrice cambia, è necessario ricalcolare l'intera struttura dati.

\subsubsection{Intuizione}Qualsiasi numero non negativo può essere rappresentato in modo univoco come somma di potenze decrescenti di due. Questa è solo una variante della rappresentazione binaria di un numero. Ad esempio $13=(1101)_2=8+4+1$. Per un certo numero $x$ possono essere al massimo $\lceil \log_2 x\rceil$ addendi.\par Con lo stesso ragionamento, qualsiasi intervallo può essere rappresentato in modo univoco come un'unione di intervalli con lunghezze che sono potenze decrescenti di due. Ad esempio $[2,14]=[2,9]\cup [10,13]\cup [14,14]$, dove l'intervallo completo ha lunghezza 13 e i singoli intervalli hanno rispettivamente lunghezza 8,4 e 1. E anche qui l'unione è composta al massimo da $\lceil \log_2(\textit{lunghzza dell'intervallo})\rceil$ molti intervalli.
\begin{lstlisting}[language=python,caption={in python3 si calcolerebbe in questo modo.}]
	import math
	math.ceil(math.log(13,2)) # risulta 4 $\rightarrow \lceil \log_2 13\rceil$
\end{lstlisting}

L'idea alla base delle tabelle sparse è quella di precalcolare tutte le risposte per query di intervallo con potenza di due lunghezze. Successivamente è possibile rispondere a una query di intervallo diverso suddividendo l'intervallo in intervalli con potenza di due lunghezze, cercando le risposte precalcolate e combinandole per ricevere una risposta completa.
\subsubsection{Precalcolo}
Si utilizzerà un array bidimensionale per memorizzare le risposte alle query precalcolate. \texttt{st[i][j]} memorizzerà la risposta per l'intervallo $[j,j+2^i-1]$ di lunghezza $2^i$. La dimensione dell'array bidimensionale sarà $(K-1)\times MAXN$, dove $MAXN$ è la lunghezza massima possibile dell'array. $K$ deve soddisfare $K\ge \lfloor \log_2 MAXN\rfloor$, perché $2^{\lfloor \log_2 MAXN\rfloor}$ è la più grande potenza di due gamme, che si deve supportare. Per array di lunghezza ragionevole ($\le 10^7$ elementi), $K=25$ è un buon rapporto qualità-prezzo.

Si importa $MAXN$ come il numero di colonne della matrice per consentire accessi consecutivi alla memoria (cache).
\begin{lstlisting}
	int st[k+1][MAXN]; //ricorda che questa notazione significa nome_matrice[righe][colonne]
\end{lstlisting} 
Perché la gamma $[j, j+2^i-1]$ di lunghezza $2^i$ si divide bene negli intervalli $[j,j+2^{i-1}-1$ e $j+2^{i-1}, j+2^i-1]$, entrambi di lunghezza $2^{i-1}$, si può generare la tabella in modo efficiente utilizzando la programmazione dinamica:
\begin{lstlisting}
	std::copy(array.begin(), array.end(), st[0]);
	
	for (int i = 1; i <= K; i++)
		for (int j = 0; j + (1 << i) <= N; j++)
			st[i][j] = f(st[i - 1][j], st[i - 1][j + (1 << (i - 1))]);
\end{lstlisting}
La funzione $f$ dipenderà dal tipo di query. Per le query con somma dell'intervallo calcolerà la somma, per le query con somma minima dell'intervallo calcolerà il minimo. La complessità temporale del precalcolo è $O(N\log N)$.

\subsubsection{Query sulla soma degli intervalli}
Per questo tipo di query, si vuole trovare la somma di tutti i valori in un intervallo. Quindi la definizione naturale della funzione $f$ è $f(x,y)=x+y$. Si può costruire la struttura dati con:
\begin{lstlisting}
	long long st[K + 1][MAXN];
	
	std::copy(array.begin(), array.end(), st[0]);
	
	for (int i = 1; i <= K; i++)
		for (int j = 0; j + (1 << i) <= N; j++)
			st[i][j] = st[i - 1][j] + st[i - 1][j + (1 << (i - 1))];
\end{lstlisting}
Per rispondere alla query di somma per l'intervallo $[L,R]$, si itera su tutte le potenze di due, partendo da quella più grande. Non appena una potenza di due $2^i$ è minore o uguale alla lunghezza dell'intervallo $(=R-L+1)$, si elabora la prima parte dell'intervallo $[L,L+2^i-1]$, e si continua con l'intervallo rimanente $[L+2*i,R]$.

\begin{lstlisting}
	long long sum = 0;
	for (int i = K; i >= 0; i--) {
		if ((1 << i) <= R - L + 1) {
			sum += st[i][L];
			L += 1 << i;
		}
	}
\end{lstlisting}
La complessità temporale per una query Range Sum è $O(K)=O(\log MAXN)$.
\subsubsection{Query minime di intervallo (RMQ)}
Sono queste le query in cui la tabella sparsa eccelle. Quando si vole calcolare il minimo di un intervallo, non imposta se si elabora un valore nell'intervallo una o due volte. Pertanto, invece di suddividere un intervallo in più intervalli, lo si può anche suddividere in soli due intervalli sovrapposti con potenza di due lunghezze. Ad esempio, si può dividere l'intervallo $[1,6]$, nelle gamme $[1,4]$ e $[3,6]$. L'intervallo minimo di $[1,6]$ è chiaramente uguale al minimo dell'intervallo minimo di $[1,4]$ e l'intervallo minimo di $[3,6]$. Quindi si può calcolare il minimo dell'intervallo $[L,R]$ con.
$$\min(st[i][L], st[i][R-2^i+1])$$ dove $i=\log_2(R-L+1)$. Ciò richiede di essere in grado di calcolare $log_2(R-L+1)$ velocemente. È possibile ottenere questo risultato precalcolando tutti i logaritmi:
\begin{lstlisting}
	int lg[MAXN+1];
	lg[1] = 0;
	for (int i = 2; i <= MAXN; i++)
		lg[i] = lg[i/2] + 1;
\end{lstlisting}

Successivamente si devono precalcolare la struttura dati della tabella sparsa. Questa volta viene definito $f$ con $f(x,y)=\min (x,y)$.
\begin{lstlisting}
	int st[K + 1][MAXN];
	
	std::copy(array.begin(), array.end(), st[0]);
	
	for (int i = 1; i <= K; i++)
		for (int j = 0; j + (1 << i) <= N; j++)
			st[i][j] = min(st[i - 1][j], st[i - 1][j + (1 << (i - 1))]);
\end{lstlisting}
Il minimo di un intervallo $[L,R]$ può essere calcolato con:
\begin{lstlisting}
	int i = lg[R - L + 1];
	int minimum = min(st[i][L], st[i][R - (1 << i) + 1]);
\end{lstlisting}
La complessità temporale per una query minima di intervallo è $O(1)$.
\subsubsection{Limitazioni}L'approccio sopra discusso supporta solo query di funzioni idempotenti. In altre parole, funziona benissimo per le query con l'intervallo minimo, ma non è possibile rispondere alle query con somma degli intervalli utilizzando questo approccio. 

Esistono strutture dati simili in grado di gestire qualsiasi tipo di funzione associativa e rispondere a query di intervallo in $O(1)$. Una si chiama \textit{tabella sparsa disgiunta}. Un'altra sarebbe l'\textit{albero quadrato}.

\subsection{Unione di insiemi disgiunti}Vengono dati diversi elementi, ognuno dei quali è un insieme separato. Una DSU ha un'operazione per combinare due insiemi qualsiasi e sarà in grado di dire in quale insieme si trova un elemento specifico. La versione classica introduce anche una terza operazione: può creare un insieme a partire da un nuovo elemento. \par Pertanto l'interfaccia di base di questa struttura dati è composta da sole tre operazioni:
\begin{itemize}
	\item \texttt{make\_set(v)}: crea un nuovo set costituito dal nuovo elemento \texttt{v};
	\item \texttt{union\_sets(a,b)}: unisce i due insiemi specificati (si trova l'insieme in cui è presente l'elemento $a$ e si trova l'insieme in cui è presente l'elemento \texttt{b});
	\item \texttt{find\_set(v)}: restituisce il rappresentante (detto anche leader) dell'insieme che contiene l'elemento \texttt{v}. Questo rappresentante è un elemento del suo insieme corrispondente. Viene selezionato in ogni set della struttura dati stessa (e può cambiare nel tempo, vale a dire dopo \texttt{union\_sets} chiamate). Questo rappresentante può essere utilizzato per verificare se due elementi fanno parte o meno dello stesso insieme. Due elementi $a$ e $b$ sono nello stesso insieme, se \texttt{find\_set(a) == find\_set(b)}. Altrimenti si trovano in set diversi.
\end{itemize} 
La struttura dati consente di eseguire ciascuna di queste operazioni in quasi $O(1)$ di tempo in media.
