	\subsubsection{Modifica dello stack}
	Per trovare l'elemento più piccolo in uno stack in $O(1)$ mantenendo il comportamento asintotico per aggiunta e rimozione di elementi, si memorizzano delle coppie. 
	\begin{figure}[tbph]

		\includegraphics[width=0.7\linewidth]{stack-min-el}
		\caption{l'elemento minimo viene calcolato ad ogni inserimento e salvato sulla seconda colonna dell'elemento in cima allo stack (in rosso).}
		\label{fig:stack-min-el}
	\end{figure}
	\paragraph{Aggiungere un elemento}
	\begin{lstlisting}[caption={aggiungere un elemento.}]
		int new_min = st.empty() ? new_elem : min(new_elem, st.top().second);
		st.push({new_elem, new_min});
	\end{lstlisting}
	\begin{lstlisting}[caption={rimuovere un elemento.}]
		int removed_element = st.top().first;
		st.pop();
	\end{lstlisting}
	\begin{lstlisting}[caption={trovare il minimo.}]
		int minimum = st.top().second;
	\end{lstlisting}Lo stack ha un comportamento LIFO.
	\subsubsection{Modifica della coda -- metodo 1}La coda ha un comportamento FIFO. Non memorizza tutti gli elementi: vengono memorizzati solo quelli necessari per determinare il minimo. La coda viene mantenuta in \textit{ordine non decrescente}\footnote{cioè $\forall i \in [0,\texttt{coda.length}()-1]\colon \texttt{coda[i]} \le \texttt{coda[i+1]}$} (il valore più piccolo sarà memorizzato nella testa). Il minimo effettivo deve sempre essere mantenuto nella coda. 
	\begin{lstlisting}[caption={implementazione della coda.}]
		deque<int> q;
	\end{lstlisting}
	\begin{lstlisting}[caption={trovare il minimo.}]
		int minimum = q.front();
	\end{lstlisting}
	Prima di aggiungere un nuovo elemento nella coda è sufficiente effettuare un ``taglio'': vengono rimossi tutti gli elementi finali della coda che sono più grandi del nuovo elemento e successivamente si aggiunge il nuovo elemento alla coda. In questo modo viene mantenuto l'ordine non decrescente e non viene perso l'elemento corrente se in qualsiasi passaggio successivo è l'elemento minimo. Tutti gli elementi che vengono rimossi non potranno mai essere primi. 

	\begin{lstlisting}[caption={aggiungere un elemento.}]
		while (!q.empty() && q.back() > new_element)
		q.pop_back();
		q.push_back(new_element);
	\end{lstlisting}
	
	Quando si vuole estrarre un elemento dalla testa, potrebbe non essere presente se è stato rimosso in precedenza con un elemento più piccolo. Pertanto quando si elimina un elemento da una coda bisogna conoscerne il valore. Se la testa della coda ha lo stesso valore, può essere rimossa in sicurezza, altrimenti non viene apportata nessuna modifica.
	\begin{lstlisting}[caption={rimuovere un elemento.}]
		if (!q.empty() && q.front() == remove_element)
		q.pop_front();
	\end{lstlisting}
	Tutte queste operazioni richiedono $O(1)$.

		\centering
		\includegraphics[width=0.7\linewidth]{queue-aggiunta-metodo1}
		\caption{effetto dell'inserimento di un nuovo elemento sulla coda.}
		\label{fig:queue-aggiunta-metodo1}
	\end{figure}
	\subsubsection{Modifica della coda -- metodo 2}
	Questa è una modifica del metodo 1. Si memorizza l'indice per ciascun elemento della coda, ricordando anche quanti elementi sono stati già aggiunti e rimossi.
\begin{lstlisting}[caption={implementazione della coda con contatori di elementi aggiunti e rimossi.}]
	deque<pair<int, int>> q;
	int cnt_added = 0;
	int cnt_removed = 0;
\end{lstlisting}
\begin{lstlisting}[caption={trovare il minimo.}]
	int minimum = q.front().first;
\end{lstlisting}
\begin{lstlisting}[caption={aggiungere un elemento.}]
	while (!q.empty() && q.back().first > new_element)
	q.pop_back();
	q.push_back({new_element, cnt_added});
	cnt_added++;
\end{lstlisting}
La modifica più rilevante è in questa porzione di codice (compararla con quella del metodo 1).
\begin{lstlisting}[caption={rimuovere un elemento.}]
	if (!q.empty() && q.front().second == cnt_removed) 
	q.pop_front();
	cnt_removed++;
\end{lstlisting}
\subsubsection{Modifica della coda -- metodo 3}
Questo è un altro metodo per trovare il minimo in $O(1)$ modificando una coda. È più complicato da implementare ma permette di memorizzare tutti gli elementi e di rimuovere un elemento dalla parte anteriore senza conoscerne il valore. L'idea è di ridurre il problema al problema degli stack, simulando una coda. Si creano due pile, \texttt{s1} e \texttt{s2}. Si aggiungono nuovi elementi allo stack \texttt{s1}. e si rimuovono da \texttt{s2}. Se in qualsiasi momento \texttt{s2} è vuoto, si spostano tutti gli elementi da \texttt{s1} a \texttt{s2} (che inverte l'ordine degli elementi). Infine, per trovare il minimo in  una coda è sufficiente trovare il minimo di entrambi gli stack. \par
Si eseguono tutte le operazioni in $O(1)$ in media (ogni elemento verrà aggiunto una volta allo stack \texttt{s1}, una volta trasferito a \texttt{s2} e una volta estratto da \texttt{s2}).
\begin{lstlisting}[caption={implementazione.}]
	stack<pair<int, int>> s1, s2;
\end{lstlisting}
\begin{lstlisting}[caption={trovare il minimo.}]
	if (s1.empty() || s2.empty()) 
		minimum = s1.empty() ? s2.top().second : s1.top().second;
	else
		minimum = min(s1.top().second, s2.top().second);
\end{lstlisting}

\begin{lstlisting}[caption={aggiungi elemento.}]
	int minimum = s1.empty() ? new_element : min(new_element, s1.top().second);
	s1.push({new_element, minimum});
\end{lstlisting}

\begin{lstlisting}[caption={rimozione di un elemento.}]
	if (s2.empty()) {
		while (!s1.empty()) {
			int element = s1.top().first;
			s1.pop();
			int minimum = s2.empty() ? element : min(element, s2.top().second);
			s2.push({element, minimum});
		}
	}
	int remove_element = s2.top().first;
	s2.pop();
\end{lstlisting}

\subsubsection{Trovare il minimo per tutti i sottoarray di lunghezza fissa}
Si supponga di ricevere un array \texttt{A} di lunghezza $N$ e un dato $M$ tale che $M\le N$. Si deve trovare il minimo di ogni sottoarray di lunghezza $M$ in questo array, cioè si trovi: $$\min_{0\le i\le M-1}A[i],\min_{1\le i \le M} A[i], \min_{2 \le i \le M+1} A[i], \dots, \min_{N-M\le i \le N-1}A[i]$$
Questo problema deve essere risolto in tempo lineare, cioè $O(n)$.
\begin{figure}[tbph]
	\centering
	\includegraphics[width=0.7\linewidth]{sottoarray-stack-coda}
	\caption{sottoarray di dimensione $M$.}
	\label{fig:sottoarray-stack-coda}
\end{figure}

È possibile utilizzare una qualsiasi delle tre code modificate per risolvere il problema. 
\begin{lstlisting}
	for(i = 0; i < N; i++){
		for(j = 0; j < M+i; j++){
			
		}
	}
\end{lstlisting}
